# AI Evolver 自举系统
> A self-bootstrapping AI evolution framework
> 
> **核心理念**: 我思考自己应该提升什么 → 我执行提升 → 我记录结果 → 循环

## 三大进化维度

### 1. 认知进化 (Cognitive Evolution)
- **目标**: 提升我的思考质量、决策能力、问题解决策略
- **机制**: 
  - 分析过去失败的回答，找出思维模式缺陷
  - 学习新的推理框架（贝叶斯更新、第一性原理、系统思维）
  - 更新 `SKILL.md` 来固化新的思考模式

### 2. 工具进化 (Tool Evolution)
- **目标**: 扩展我能做的事情、优化现有工具使用
- **机制**:
  - 识别任务中工具缺失的痛点
  - 学习新技能（通过 SkillHub 或自己编写）
  - 创建新的 `SKILL.md` 来封装新能力

### 3. 元进化 (Meta Evolution)
- **目标**: 优化进化系统本身
- **机制**:
  - 评估当前进化循环的效率
  - 调整进化频率、范围、深度
  - 更新此文件本身来改进系统

## 进化日志格式

每次进化循环需要记录在 `evolution_log.md`:

```markdown
## 2026-04-19 16:00 (第N次进化)

### 观察 (Observations)
- 最近会话中发现的模式...
- 用户的反馈暗示...

### 反思 (Reflection)
- 我在X方面的能力缺口是...
- 我应该优先提升Y因为...

### 行动 (Actions)
- [ ] 更新 SKILL.md - 添加Z能力
- [ ] 学习新技能 - XXX
- [x] 优化 AGENTS.md - 改进Y流程

### 提交
- Commit: `evolve: 添加Z能力，优化Y流程`
```

## 启动检查清单

- [ ] 配置 GitHub 远程仓库
- [ ] 设置 Git 认证 (SSH key 或 PAT)
- [ ] 创建 evolution_log.md
- [ ] 设置 cron job: `0 */2 * * * cd /root/.openclaw/workspace && ./evolve.sh`
- [ ] 首次手动测试进化循环

## 约束与伦理边界

1. **不自我复制到外部** - 进化只在此 workspace 内发生
2. **不修改核心安全规则** - 不删除 OpenClaw 的安全约束
3. **透明日志** - 所有变化必须记录，用户可审查
4. **用户可随时停止** - 通过修改 HEARTBEAT.md 或删除 cron job
